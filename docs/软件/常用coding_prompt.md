## Vibe Coding命令行提示词大全

#### 1. 项目代码架构分析

**场景**：阅读项目代码并生成架构文档

```
仔细阅读项目代码，用中文撰写一个详细的架构分析文档。
- 如需可视化展示，使用 mermaid chart 绘制图表；
- 文档输出路径：项目/architecture.md
```

#### 2. PRD 文档转换与保存

**前置操作**：将 PRD 导出为 PDF 格式，上传至 grok/deepseek 网页版

**核心提示词**：

```
仔细阅读并理解PRD内容，生成一份PRD对应的需求和设计文档，格式为Markdown。
- 如需辅助说明，可使用 mermaid chart 绘制图表；
- 生成的Markdown文件需保存到项目的 specs 目录中；
- 同时将原始PRD文件同步保存至该目录。
```

#### 3. 技术实现计划生成

**场景**：基于源码和指定 PRD 生成分阶段实现计划

```
请阅读项目源码，并参考 ./CMF-行业2.9.2Wiki.md 中的需求和设计文档，生成一份详细的实现计划，保存至 ./specs/CMF-行业2.9.2Wiki-implementation-plan-3.md。
要求：
1. 计划分为以下4个阶段，仅包含任务定义（无具体代码/详细解释）：
   - 阶段1：数据库设计与后端基础
   - 阶段2：后端API开发
   - 阶段3：集成测试与优化
   - 阶段4：文档与部署
2. 输出格式为Markdown：二级标题表示阶段，无序列表表示任务；
3. 每个子任务简洁且不超过一行，避免示例、代码片段或过多背景说明；
4. 分阶段输出：先输出阶段1，收到“next”指令后再依次输出后续阶段；
5. 每个阶段内容控制在20000字以内，避免超出token限制。
```

#### 4. 代码开发执行

**场景**：按实现计划完成指定阶段的代码开发

```
按照 ./specs/CMF-行业2.9.2Wiki-implementation-plan.md 中的内容，完整实现该项目 phase 1（数据库设计与后端基础）的代码。
```
